# Requirements Document

## Introduction

When a user opens a shared grocery list link on a mobile device for the first time, the app should display a banner prompting them to install the PWA via "Add to Home Screen." On Android and other browsers that support the `beforeinstallprompt` event, the banner includes an Install button that triggers the native install flow. On iOS Safari (which lacks `beforeinstallprompt`), the banner shows manual instructions for using the Share menu. The banner is only shown once per device, only on mobile, and only when the app is not already running in standalone mode.

## Glossary

- **Install_Prompt_Banner**: A dismissible UI banner displayed at the bottom of the app viewport that informs the user about installing the PWA and provides an action to do so.
- **App**: The Grocery List PWA application.
- **Shared_List_URL**: A URL containing an encoded grocery list in the `?list=` query parameter, generated by the share controller.
- **Standalone_Mode**: The display mode when the PWA is launched from the home screen (`display-mode: standalone`), as opposed to running inside a browser tab.
- **BeforeInstallPrompt_Event**: A browser event fired when the PWA meets installability criteria, allowing the app to programmatically trigger the native install dialog. Supported on Chromium-based browsers and Firefox on Android; not supported on iOS Safari.
- **Dismissal_Flag**: A key stored in localStorage that records whether the user has dismissed or acted on the Install_Prompt_Banner, preventing it from appearing again.
- **Mobile_Device**: A device whose user agent matches a mobile platform (Android, iPhone, iPad) or whose viewport width and touch capability indicate a handheld device.

## Requirements

### Requirement 1: Display Install Prompt on Shared List Import

**User Story:** As a user who received a shared grocery list link, I want to see a message about saving the app to my home screen, so that I can quickly access the app again later.

#### Acceptance Criteria

1. WHEN a Shared_List_URL is opened on a Mobile_Device AND the App is not running in Standalone_Mode AND no Dismissal_Flag exists in localStorage, THE Install_Prompt_Banner SHALL be displayed after the shared list import flow completes.
2. WHILE the App is running in Standalone_Mode, THE Install_Prompt_Banner SHALL remain hidden.
3. WHILE the App is running on a non-mobile device, THE Install_Prompt_Banner SHALL remain hidden.
4. WHEN a Dismissal_Flag exists in localStorage, THE Install_Prompt_Banner SHALL remain hidden regardless of other conditions.

### Requirement 2: Banner Content and Layout

**User Story:** As a mobile user, I want the install banner to clearly explain what "Add to Home Screen" does and how to do it, so that I understand the benefit and can act on it.

#### Acceptance Criteria

1. THE Install_Prompt_Banner SHALL display a message explaining that the user can save the grocery list app to the home screen for quick access.
2. THE Install_Prompt_Banner SHALL include a dismiss button that allows the user to close the banner.
3. THE Install_Prompt_Banner SHALL be rendered at the bottom of the viewport so it does not obscure the imported list content.
4. THE Install_Prompt_Banner SHALL have a role of "banner" and the dismiss button SHALL have an accessible label of "Dismiss install prompt".

### Requirement 3: Native Install Flow on Supported Browsers

**User Story:** As an Android user, I want to tap an Install button that directly triggers the browser's install dialog, so that I can add the app to my home screen without extra steps.

#### Acceptance Criteria

1. WHEN the BeforeInstallPrompt_Event has been captured, THE Install_Prompt_Banner SHALL display an "Install" button.
2. WHEN the user activates the "Install" button, THE App SHALL call `prompt()` on the stored BeforeInstallPrompt_Event to trigger the native install dialog.
3. WHEN the user accepts the native install dialog, THE App SHALL set the Dismissal_Flag in localStorage and hide the Install_Prompt_Banner.
4. WHEN the user dismisses the native install dialog, THE Install_Prompt_Banner SHALL remain visible so the user can try again later.

### Requirement 4: Manual Install Instructions on iOS

**User Story:** As an iOS Safari user, I want to see step-by-step instructions for adding the app to my home screen, so that I can install it even though my browser does not support automatic install prompts.

#### Acceptance Criteria

1. WHEN the BeforeInstallPrompt_Event has not been captured AND the Mobile_Device is running iOS Safari, THE Install_Prompt_Banner SHALL display instructions to tap the Share icon and then "Add to Home Screen".
2. THE Install_Prompt_Banner SHALL not display an "Install" button when the BeforeInstallPrompt_Event is unavailable.

### Requirement 5: Dismissal Persistence

**User Story:** As a returning user, I do not want to see the install prompt again after I have already dismissed it, so that the app does not repeatedly interrupt my experience.

#### Acceptance Criteria

1. WHEN the user activates the dismiss button on the Install_Prompt_Banner, THE App SHALL set the Dismissal_Flag in localStorage.
2. WHEN the user activates the dismiss button, THE Install_Prompt_Banner SHALL be removed from the DOM.
3. WHEN the App loads and a Dismissal_Flag exists in localStorage, THE App SHALL not create the Install_Prompt_Banner.

### Requirement 6: Detect Mobile and Standalone Context

**User Story:** As a developer, I want reliable detection of mobile devices and standalone mode, so that the install prompt only appears in the correct context.

#### Acceptance Criteria

1. THE App SHALL detect Standalone_Mode by checking the `display-mode: standalone` CSS media query and the `navigator.standalone` property (for iOS).
2. THE App SHALL detect a Mobile_Device by checking the user agent string for mobile platform identifiers (Android, iPhone, iPad) combined with touch capability via `navigator.maxTouchPoints`.
3. IF the detection logic cannot determine the device type, THEN THE App SHALL default to not showing the Install_Prompt_Banner.
